使用hexo搭建博客，并且部署到github

2019-07-28
  • 记录

前言

最近突然想搭建一个自己的技术博客，想了想觉得可以使用hexo搭建一个静态的博客，部署在github上，再弄个域名，不用去买服务器，也不用后台。费了一番功夫，也算是弄好了。然后顺便把这个过程记录下来，这也就是我的第一篇文章。

步骤

1.安装git （这个我就不细说了）
2.安装nodejs
3.安装hexo
4.搭建博客
5.写一篇文章
6.部署到github（你需要有一个github账号）

安装nodejs

去官网下载最新版本，安装

安装好之后可以验证一下：

node -v

然后安装cnpm（可以不使用cnpm）：

npm install -g cnpm --registry=https://registry.npm.taobao.org

安装hexo

使用命令：

cnpm install -g hexo-cli

如果没有安装cnpm，就用npm：

npm install -g hexo-cli

安装完之后可以验证一下：

hexo -v

搭建博客

直接在某个位置新建一个文件夹，名字随意，比如blog

也可以在命令行窗口使用 mkdir blog 新建一个文件夹

然后进到那个文件夹目录：

cd blog

再使用hexo生成一个博客：

hexo init

这样就成功在本地搭建了一个博客，可以启动看看：

hexo s

ctrl+c可以停止

这样就启动好了

打开http://localhost:4000

新建一篇文章

在blog目录下

hexo new "我的文章"

这个是基于markdown格式的,你可以去看看markdown语法。

然后重新生成一下(在blog的根目录)

hexo clean
hexo g

刷新一下网页就可以看到你的文章了

最后我们可以把它部署到github上

部署到github

先登录到你的github账号
再新建一个仓库

仓库名字要是 你的github昵称.github.io
如 umrm.github.io
描述可以随便写一个   
最后再create，就完成了。

现在你有了一个空仓库

你需要安装一个Git的插件

cnpm install –save hexo-deployer-git

再设置一下

在blog目录下打开配置文件_config.yml
找到deploy
把repo改成你刚建的那个仓库，别忘了空格

deploy:
  type: git
  repo: https://github.com/umrm/umrm.github.io.git
  branch: master

可以部署了

使用hexo d命令来部署到github上

hexo d

第一次会需要你输入github的账号和密码登录一下，以后就不用了

成功之后就可以在浏览器上可以直接通过你的仓库名进行访问

hexo有一个默认的主题，你还可以在网上去下载别的主题
